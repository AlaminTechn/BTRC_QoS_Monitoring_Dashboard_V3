Recommendation: Metabase for BTRC QoS Dashboard

Strengths & Weaknesses
Apache Superset - Strengths
  • 50+ visualization types including advanced charts (Sunburst, Treemap, Heatmap)
  • SQL Lab is excellent for data exploration and ad-hoc queries
  • Large open-source community (65k+ GitHub stars)
  • Apache Foundation backing ensures long-term maintenance
  • Good for data engineering teams with SQL expertise
  • Native cross-filter support via emit_filter
Apache Superset - Weaknesses (BTRC Context)
  • Custom GeoJSON requires manual file replacement inside Docker
  • country_map only supports Level-1 subdivisions (Divisions only, no Districts)
  • deck.gl maps broken in Chrome (WebGL luma.gl null error)
  • API-created native filters broken ("Apply Filters" stays disabled)
  • Complex Docker setup: 5 containers (web, worker, beat, Redis, Postgres)
  • Higher memory footprint (~2-3 GB RAM)
  • Steep learning curve for non-technical BTRC officers
  • Time Range filter API has empty target issues
Metabase - Strengths
  • Custom GeoJSON upload via Admin UI - supports Division AND District maps
  • Dropdown filters with static-list values work reliably via API
  • Simple Docker deployment: 1 container + shared database
  • Modern, intuitive UI accessible to non-technical users
  • Full REST API for dashboard automation and CI/CD
  • Low memory footprint (~1-1.5 GB RAM)
  • Click-to-drill with custom destinations for hierarchy navigation
  • Excellent documentation and active community
  • Dashboard tabs with per-tab card layout
  • Date picker filters work correctly for time range filtering
Metabase - Weaknesses
  • Fewer chart types (20 vs 50+) - no Sunburst, Treemap natively
  • SQL editor less powerful than Superset SQL Lab
  • No true map drill-down (click region to zoom) - uses filter workaround
  • AGPL license (more restrictive than Apache 2.0 for embedding)
  • Smaller community than Superset (40k vs 65k stars)
